### 2.2 Elo 赔率算法

```
有效主队Elo = Elo_Home + 100（主场优势）
P_Home = 1 / (1 + 10^((Elo_Away - 有效主队Elo) / 400))
P_Draw = 0.28 × e^(-diff² / 180000)
odds = max(1.05, (1/P) × 0.95)  // 5% 庄家抽水
```

赛后更新：K=32（小组赛）/ 40（淘汰赛），胜者加分输者减分。

## 五、补齐功能（Phase 7-11）

### 5.1 比赛倒计时

`CountdownTimer` 组件，精确到秒实时更新，集成到赛事卡片（compact 模式）和比赛详情页（完整模式）。

### 5.2 个人中心 + 密码修改

`/profile` 页展示用户信息，`PUT /api/auth/change-password` 验证旧密码后 bcrypt 更新。

### 5.3 积分榜

`GET /api/standings` — 遍历 finished 小组赛，计算 12 组积分榜。排序规则：积分 → 净胜球 → 进球数。

### 5.4 投注限额动态读取

- 前端：`GET /api/config/public` 读取 min/max，输入框红色高亮实时校验
- 后端：`placeBet` 中二次拦截，拒绝超限投注

### 5.5 余额实时校验

投注金额超过余额时红色提示 + 按钮置灰 + 显示具体余额不足信息。

### 5.6 淘汰赛对阵图

`/bracket` 页，按阶段（32强→16强→8强→半决赛→三四名→决赛）分组展示淘汰赛对阵卡片，含比分和晋级标注。

### 5.7 自动状态流转

`autoTransitionStatuses()` — 每次查询比赛时自动检测：
- `scheduled` 且 match_date < now → `live`
- `live` 且已过 90 分钟 → `finished` + 随机比分

### 5.8 消息通知

`GET /api/notifications` — 返回最近 30 条结算通知。Header 铃铛图标显示未读数红点（1 小时内结算 = 未读）。

### 5.9 真实比分模拟（Elo + 泊松分布）

Live→finished 阶段不再纯随机，改为基于 Elo 差距的泊松分布模型：

```
预期进球_Home = LeagueAvg × exp((Elo_Home - AvgElo) / Scale)
预期进球_Away = LeagueAvg × exp((Elo_Away - AvgElo) / Scale)

泊松采样 P(X=k) = (λ^k × e^(-λ)) / k!
各生成 0~8 球概率分布，加权采样产出最终比分
```

强队预期进球更高，弱队更低，更符合真实足球比分分布。

### 5.10 下注确认弹窗

投注前弹出 Modal 二次确认：展示比赛信息、投注类型、金额、赔率、潜在回报，用户确认后才真正提交。

### 5.11 数据导出（CSV）

`GET /api/admin/export/bets` — 管理员导出全部投注记录为 CSV 文件，包含用户/比赛/投注类型/金额/赔率/状态/时间等字段。

### 5.12 优雅退出 + 一键启动

- `SIGINT`/`SIGTERM` 自动保存 SQLite 到磁盘
- `start-all.bat`：一键启动三服务
- `build-all.bat`：生产构建前/后台

## 七、第二轮补齐（Phase 12-15）

### 7.1 批量生成小组赛赛程

`POST /api/admin/generate-schedule` — 为全部 12 个组自动生成双循环赛程：
- 每组 4 队：第1轮(A1vA2, A3vA4)、第2轮(A1vA3, A2vA4)、第3轮(A1vA4, A2vA3)
- 每天 2 场（17:00 + 20:00），3 轮 × 4 组计算每天推进
- 共 72 场，自动计算赔率
- 管理端 `MatchesPage` 添加"批量生成小组赛"按钮

### 7.2 Elo + 泊松分布比分模拟

`simulateScore(homeElo, awayElo)` — 替换纯随机比分：
- 基于 Elo 估算预期进球：`λ = LeagueAvg × exp((Elo - AvgElo) / Scale)`
- 泊松采样生成 0-15 球概率分布
- 主场优势 +15% 预期进球
- 强队预期 5+ 球，弱队预期 1 球左右，分布更符合真实足球

### 7.3 下注二次确认弹窗

- 点击"投注 X 币"先弹出确认面板（比赛/类型/金额/赔率/回报）
- 再点"确认下注"才真正提交
- 投注成功自动收起确认面板

### 7.4 投注数据导出 CSV

`GET /api/admin/export/bets` — 管理员一键导出全量投注记录：
- 字段：ID/用户名/邮箱/主队/客队/类型/金额/赔率/回报/状态/时间
- UTF-8 BOM 兼容 Excel 直接打开
- 管理端 `BetsPage` 添加"导出 CSV"按钮

## 十、第五轮补齐（Phase 25）：多盘口赔率系统

### 盘口类型

| 盘口 | market_type | 计算方式 | 显示 |
|------|------------|---------|------|
| 全场胜负 | `full_time` | Elo + 主场优势 + 平局高斯 | 主胜/平局/客胜 三按钮 |
| 上半场 | `first_half` | Elo（减半主场优势）+ 更高平局率 | 同上 |
| 下半场 | `second_half` | 同上半场 | 同上 |
| 正确比分 | `correct_score` | 泊松分布 × 20 种最可能比分 | 滚动网格（如 1-0@7.5） |
| 点球 | `penalty` | Elo 差距越小概率越高（~15-25%） | 有/无 两按钮 |
| 角球 9.5 | `corners` | Elo 强弱 → 预期角球 → 泊松过线率 | 大于/小于 9.5 |

### 数据库变更

- `odds` 表：market_type 列 + UNIQUE(match_id, market_type) + odds_data JSON 列
- `bets` 表：market_type 列
- 自动迁移旧数据库

### 前端变更

- `MatchDetailPage`：盘口标签页切换（6 个 tab）
- 不同盘口不同 UI（三按钮 / 比分网格 / 是否 / 大小）
- 投注单显示当前选中盘口 + 选项 + 赔率
- API：`GET /api/odds/match/:id/all` 返回全部盘口
